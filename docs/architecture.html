The toolkit · what each piece does
01

BlackHole

Open-source · macOS · brew install

A free virtual audio cable that splits system audio so the laptop can hear the meeting and you still hear it through your headphones.

Without it, no Teams or Meet call can be captured at all.

02

Apple SpeechAnalyzer

macOS 26 native · Swift sidecar binary

On-device speech-to-text built into the OS. A small Swift binary streams Int16 PCM in and emits transcript JSONL out.

Customer voice never leaves the laptop. Free, fast, and accurate.

03

mlx-whisper · WhisperWorker

Apple MLX · whisper-large-v3-turbo · async sidecar

A second on-device STT pass. Apple's sidecar gives us live, low-latency draft text; this worker re-transcribes each finalized segment from a 60-second mic+system rolling buffer and broadcasts a transcript_update to upgrade the text in place. Question detection is gated on whisper_final so the model reasons over the upgraded text, not the draft. Drained before summary generation.

Apple is fast but loose with proper nouns and product names; Whisper is slower but accurate. This combination gives a live transcript and a high-quality one — without making the user wait.

04

Resemblyzer

Python · CPU voice embeddings

Open-source voice-fingerprint model. Tags each segment with "Me" or "Speaker N" using a centroid-clustering approach.

Lets the AI react differently when you talk vs the customer asking.

05

Claude Opus 4.7

Salesforce AI Model Gateway · 4 prompts

The brain. Maintains a structured meeting state, detects real customer questions, synthesizes grounded answers, and writes the post-meeting summary.

One model, four roles — counts toward existing AI budget.

06

Slack + Web research

Slack MCP · DuckDuckGo · page fetch

The answer engine searches internal Slack (product, engineering, enablement channels) and the public web — help docs, release notes, Trailhead — then ranks sources by a strict tier hierarchy before answering.

Internal context + official docs. Stops the AI from guessing; every answer cites its source.

07

FastAPI + WebSocket

Python 3.13 · uvicorn · localhost:8765

The plumbing that wires audio capture, transcription, detection, and answers together — and pushes live updates to the browser.

No infra to deploy. python -m app.main is the whole launch command.

08

Browser sidebar

Vanilla HTML / CSS / JS · second monitor

Live transcript on the left, Q&A cards with cited URLs on the right. Speakers are renameable inline as you identify them. Transcript lines stamped with data-segment-id so the Whisper worker can upgrade them in place.

Nothing for the user to install — just localhost:8765.

09

Google Docs export

Claude MCP · google-adc server

After the meeting, one click hands the summary to Claude's existing MCP integration, which creates a fresh Google Doc.

Customer-ready notes land in your Drive automatically — no copy-paste.

Audio leaves the meeting → captured locally → transcribed on-device → labeled by speaker → fed to Claude, which spots the customer's question, researches it on the web, and writes the answer in a sidebar before the customer has finished talking. After the call, the same brain writes the summary and pushes it to Google Docs.
